# Docs Page Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build a Liveline-style single-page documentation site for orderkit with live interactive demos.

**Architecture:** Next.js app router in `site/` directory (not `docs/` — avoids GitHub Pages collision). Each page section is a `"use client"` component with its own CSS module. The page imports orderkit from `../src` via tsconfig paths. PredictionMarketSim extracted as a shared module.

**Tech Stack:** Next.js 15, React 19, CSS Modules, shiki (syntax highlighting), Vercel deployment

**Spec:** `docs/superpowers/specs/2026-03-27-docs-page-design.md`

### Task 4: GettingStarted section

**Files:**
- Create: `site/components/GettingStarted.tsx`
- Create: `site/components/CodeBlock.tsx`

- [ ] **Step 1: Build CodeBlock.tsx**

Server component that takes `code` string and `language` string. Uses shiki to highlight at build time (no client JS). Returns a `<div className="code-block">` with highlighted HTML.

**Note on client components:** `CodeBlock` is a server component and cannot be imported into `"use client"` components directly. For sections that are client components AND need code snippets (LiveExchange, DepthModes, EdgeCases), render the `<CodeBlock>` OUTSIDE the client component boundary in `page.tsx` by splitting those sections: the demo part is the client component, the code block is rendered adjacent to it in the server-rendered page. Alternatively, use static `<pre>` blocks with inline styles for the few code snippets inside client components.
